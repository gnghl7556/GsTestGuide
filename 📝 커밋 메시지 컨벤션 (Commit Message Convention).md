# 📝 커밋 메시지 컨벤션 (Commit Message Convention)

본 문서는 프로젝트의 커밋 메시지 작성 규칙을 정의합니다. 일관된 커밋 메시지는 프로젝트 히스토리를 명확하게 하고, 협업 시 변경 사항을 쉽게 파악할 수 있도록 도와줍니다.

---

## 📌 커밋 메시지 형식

```
<이모티콘> <타입>: <제목>

<본문 (선택사항)>

<꼬리말 (선택사항)>
```

### 예시
```
✨ feat: 사용자 로그인 기능 추가

- JWT 토큰 기반 인증 구현
- 로그인 폼 컴포넌트 생성
- 로그인 API 연동

Resolves: #123
```

---

## 🏷️ 커밋 타입 및 이모티콘

| 이모티콘 | 타입 | 설명 |
|:---:|:---|:---|
| ✨ | `feat` | 새로운 기능 추가 |
| 🐛 | `fix` | 버그 수정 |
| 📝 | `docs` | 문서 수정 (README, 주석 등) |
| 💄 | `style` | 공백, 세미콜론 등 스타일 수정 (코드 동작에 영향 없음) |
| ♻️ | `refactor` | 코드 리팩토링 (기능 변경 없이 코드 구조 개선) |
| ⚡ | `perf` | 성능 개선 |
| ✅ | `test` | 테스트 추가 또는 수정 |
| 🔧 | `chore` | 빌드 과정 또는 보조 기능 수정 (패키지 매니저 설정 등) |

## 📋 작성 규칙

### 1. 제목 (Subject)
- 제목은 **50자 이내**로 작성합니다.
- 제목 끝에 **마침표(.)를 붙이지 않습니다.**
- **명령문** 형태로 작성합니다. (예: "추가", "수정", "삭제")
- 타입과 제목 사이에는 **콜론(:)과 공백**을 사용합니다.

### 2. 본문 (Body)
- 본문은 **선택사항**입니다.
- **무엇을**, **왜** 변경했는지 설명합니다.
- 제목과 본문 사이에는 **빈 줄**을 넣습니다.
- 한 줄은 **72자 이내**로 작성합니다.

### 3. 꼬리말 (Footer)
- 꼬리말은 **선택사항**입니다.
- **이슈 트래커 ID**를 참조할 때 사용합니다.
- **Breaking Changes**가 있을 경우 명시합니다.

## 💡 커밋 메시지 예시

### ✨ feat (새로운 기능 추가)
```
✨ feat: 회원가입 이메일 인증 기능 추가
```

### 🐛 fix (버그 수정)
```
🐛 fix: 로그인 시 비밀번호 검증 오류 수정
```

### 📝 docs (문서 수정)
```
📝 docs: README에 설치 방법 추가
```

### 💄 style (스타일 수정)
```
💄 style: 코드 포맷팅 및 불필요한 공백 제거
```

### ♻️ refactor (코드 리팩토링)
```
♻️ refactor: 사용자 서비스 로직 분리
```

### ⚡ perf (성능 개선)
```
⚡ perf: 이미지 로딩 지연 처리로 초기 로딩 속도 개선
```

### ✅ test (테스트 추가)
```
✅ test: 로그인 API 단위 테스트 추가
```

### 🔧 chore (빌드/보조 기능 수정)
```
🔧 chore: ESLint 설정 파일 업데이트
```

## 🛠️ 적용 방법

### 방법 1: Git 커밋 템플릿 설정 (권장)

프로젝트 루트에 있는 `.gitmessage` 파일을 커밋 템플릿으로 설정합니다.

```bash
# 프로젝트별 설정 (해당 저장소에서만 적용)
git config commit.template .gitmessage

# 전역 설정 (모든 저장소에 적용)
git config --global commit.template /path/to/.gitmessage
```

### 방법 2: 직접 커밋 메시지 작성

```bash
git commit -m "✨ feat: 새로운 기능 설명"
```

### 방법 3: VS Code 확장 프로그램 활용

- **Git Commit Message Editor**: 커밋 메시지 작성을 도와주는 확장 프로그램
- **Gitmoji**: 이모티콘 기반 커밋 메시지 작성 지원

## 📊 빠른 참조 표

| 상황 | 타입 | 예시 |
|:---|:---:|:---|
| 새 기능을 추가했을 때 | ✨ feat | `✨ feat: 다크모드 기능 추가` |
| 버그를 고쳤을 때 | 🐛 fix | `🐛 fix: 날짜 표시 오류 수정` |
| 문서를 수정했을 때 | 📝 docs | `📝 docs: API 문서 업데이트` |
| 코드 스타일만 변경했을 때 | 💄 style | `💄 style: 들여쓰기 수정` |
| 코드를 리팩토링했을 때 | ♻️ refactor | `♻️ refactor: 함수 분리` |
| 성능을 개선했을 때 | ⚡ perf | `⚡ perf: 쿼리 최적화` |
| 테스트를 추가했을 때 | ✅ test | `✅ test: 유닛 테스트 추가` |
| 빌드/설정을 변경했을 때 | 🔧 chore | `🔧 chore: 의존성 업데이트` |
