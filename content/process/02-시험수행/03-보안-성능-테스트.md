## 설명
기능 테스트와 병행하여, 제품 유형과 PL 지시에 따라 보안 테스트(Invicti)와 성능 테스트를 조건부로 수행하는 단계. 보안·성능 테스트는 시험마다 수행 여부와 시점이 달라지므로 PL과 사전 협의하여 진행한다.

## 체크포인트
- [ ] (조건부) 웹 제품인 경우, Invicti 보안 테스트 도구를 예약하고 스캔을 수행했는가? 스캔 결과(HTML/PDF)를 저장했는가?
- [ ] (조건부) PL이 성능 테스트 시나리오를 제공한 경우, 시나리오에 따라 성능 테스트를 수행하고 결과를 저장했는가?
- [ ] 보안·성능 테스트 결과를 결함 보고(EXEC-04)에 포함할 시점(1차/2차)을 결정했는가?

## 증빙 예시
- Invicti 보안 스캔 결과 보고서 (HTML, PDF — Executive Summary 포함)
- 성능 테스트 보고서 (PL 제공 시나리오 기반 측정 결과)
- 성능 모니터링 로그 (perfmon, top 등 시스템 자원 사용량 기록)

## 합격 기준
해당 조건에 해당하는 보안·성능 테스트가 수행되고 결과가 저장된 상태이다. 해당 조건에 해당하지 않는 경우 이 항목은 생략한다.

## TIP
> Invicti 보안 테스트를 우선 수행하는 이유: 제품이 완전히 뻗거나 기능 장애가 발생할 수 있어 중대 결함을 먼저 발견하기 위함입니다.
> 보안 도구 결과를 1차에 포함할지 2차에 포함할지는 결함 개수와 심각도를 고려하여 결정하세요.
> 성능 테스트 시나리오는 PL이 1차 결함 리포트 제공 날짜에 제공합니다.
> Invicti 스캔 중에는 해당 제품에 대한 다른 테스트를 수행할 수 없으므로 시간 배분에 유의하세요.
