## 설명
패치 완료 후, 결함 수정 여부를 확인하고 파생 결함을 탐색하며, 패치 후 RawData를 확보한 뒤 회귀 리포트를 작성하여 시험수행을 마무리하는 단계. 보안·성능 재테스트가 필요한 경우 이 단계에서 함께 수행한다.

## 체크포인트
- [ ] 업체가 회신한 결함 리포트의 결함별 '업체 응답' 항목을 검토했는가? [ref: 결함 리포트(완성본 템플릿)]
- [ ] 패치 전후 비교를 통해 결함의 실제 수정 내역을 확인·검증했는가?
- [ ] 패치로 인한 파생(회귀) 결함이 발생하지 않았는지 확인했는가?
- [ ] 패치 후 RawData(스크린샷/영상)를 기능별로 정리·저장했는가?
- [ ] (조건부) 패치 후 보안·성능 재테스트가 필요한 경우, 수행하고 결과를 저장했는가?
- [ ] 회귀 리포트를 작성하고, PL 검토 후 업체에 발송했는가? [ref: 결함 리포트(완성본 템플릿)]

## 증빙 예시
- 1차 회귀 리포트 (업체 제공용, 결함 수정 확인 결과 포함)
- (필요시) 2차 회귀 리포트 (업체 제공용)
- 패치 전후 비교 캡처 (동일 기능의 수정 전·후 화면)
- 패치 후 RawData (기능별 스크린샷/영상, 폴더별 정리)
- (해당 시) 패치 후 보안성 테스트 보고서 (HTML, PDF)
- (해당 시) 패치 후 성능 테스트 보고서

## 합격 기준
패치 후 결함 수정 확인 및 회귀 리포트가 PL 검토를 거쳐 업체에 정상적으로 제공되었다.

## 참조 문서
| 문서명 | 종류 | 설명 |
|--------|------|------|
| 결함 리포트(완성본 템플릿) | file | 회귀 리포트 작성 시 양식 참고용 완성 샘플 |
| 회귀 리포트(샘플) | file | 회귀 테스트 결과 작성용 샘플 |

## TIP
> 이 단계 이후 발견된 결함은 수정 불가하며 인증위에 그대로 상정됩니다.
> 보안·성능 재테스트는 파생 결함이 없는 경우에만 수행합니다. 파생 결함이 있으면 2차 패치 이후로 연기됩니다.
> RawData는 패치 전과 동일한 기능별 구조로 저장하여 전후 비교가 용이하도록 하세요.
