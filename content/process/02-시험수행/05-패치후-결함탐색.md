## 설명
패치 후 보안 도구 재수행, 결함 수정 확인, 파생 결함 확인, 성능 재테스트를 수행하고 패치 후 RawData를 저장한다.

## 체크포인트
- [ ] 패치 완료 후 보안 테스트 도구(Invicti)를 재예약/수행했는가?
- [ ] 보안 테스트 중 업체의 1차/2차 리포트 응답을 검토했는가?
- [ ] 결함 수정 여부와 파생 결함을 확인했는가?
- [ ] 파생 결함 유무에 따라 성능 테스트 진행 여부를 결정했는가?
- [ ] 패치 후 캡처(RawData)를 기능별로 저장했는가?

## 증빙 예시
- 패치 후 RawData (기능별 캡처 이미지)
- 보안 테스트 재수행 결과
- 결함 수정 확인 증빙

## 합격 기준
패치 후 보안·성능 재테스트 및 결함 수정 확인이 완료되어 회귀 리포트 작성이 가능한 상태이다.

## TIP
> 보안 테스트 도구 사용 중에는 제품 조작이나 다른 업무를 진행할 수 없습니다.
> 보안 테스트 중 업체 응답을 검토하여 회귀 테스트 방향을 미리 파악하세요.
> **[Case A] 파생 결함 없음**: 패치 후 성능 테스트 수행 → 회귀 리포트에 포함
> **[Case B] 파생 결함 있음**: 성능 테스트는 2차 패치 이후로 연기
