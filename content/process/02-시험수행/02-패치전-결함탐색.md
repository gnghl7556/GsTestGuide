## 설명
기능 테스트, 보안성 테스트(Invicti), 성능 테스트를 수행하여 결함을 발견하고, 패치 전 RawData를 저장한다.

## 체크포인트
- [ ] 기능 리스트 기반 기능 테스트를 수행했는가? [ref: 매뉴얼]
- [ ] 품질특성별 결함을 탐색하고 있는가? [ref: 결함 분류 기준표]
- [ ] 웹 제품인 경우 보안 테스트 도구(Invicti)를 예약 및 수행했는가?
- [ ] PL이 제공한 시나리오에 따라 성능 테스트를 수행했는가?
- [ ] 패치 전 캡처(RawData)를 기능별로 저장했는가? [ref: 결함 리포트(빈 템플릿)]

## 증빙 예시
- 1차 결함리포트
- 2차 결함리포트
- 보안성 테스트 보고서(Html, Pdf)
- 성능 보고서
- 패치 전 RawData(제품 기능 이미지)

## 합격 기준
기능·보안·성능 테스트가 진행되어 결함 리포트 작성이 가능한 상태이다.

## 참조 문서
| 문서명 | 종류 | 설명 |
|--------|------|------|
| 결함 리포트(빈 템플릿) | file | 결함 기록용 빈 양식 샘플 |
| 매뉴얼 | file | 기능 테스트 시 제품 동작 확인 기준 |
| 결함 분류 기준표 | file | 품질특성별 결함 유형 및 등급 분류 기준 |

## TIP
> 보안 테스트 도구(Invicti)를 우선 수행하는 이유: 제품이 완전히 뻗거나 기능 장애가 발생할 수 있어 중대 결함을 먼저 발견하기 위함입니다.
> 보안 도구 결과를 1차에 포함할지 2차에 포함할지는 결함 개수와 심각도를 고려하여 결정하세요.
> 성능 테스트 시나리오는 PL이 1차 결함 리포트 제공 날짜에 제공합니다.
