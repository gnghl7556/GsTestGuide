## 설명
기능 리스트 작성 후, 기능 리스트를 기반으로 품질특성별 기능 테스트를 수행하고 결함을 발견·분류하며, 패치 전 RawData를 확보하는 단계. 이 단계에서 확보한 결함 목록과 RawData가 결함 보고(EXEC-04)의 근거가 된다.

## 체크포인트
- [ ] 기능 리스트를 기반으로 품질특성별(기능적합성, 사용성, 신뢰성 등) 기능 테스트를 수행하고 발견한 결함을 기록했는가? [ref: 사용자 매뉴얼]
- [ ] 발견한 결함을 결함 분류 기준표에 따라 품질특성·심각도별로 분류했는가? [ref: 결함 분류 기준표]
- [ ] 패치 전 RawData(스크린샷/영상)를 기능별로 정리·저장했는가? [ref: 결함 리포트(빈 템플릿)]

## 증빙 예시
- 패치 전 RawData(제품 기능 이미지)
- 결함 기록 목록

## 합격 기준
기능 리스트 기반 품질특성별 기능 테스트가 수행되어 결함 리포트 작성이 가능한 상태이다.

## 참조 문서
| 문서명 | 종류 | 설명 |
|--------|------|------|
| 사용자 매뉴얼 | file | 기능 테스트 시 제품 동작 확인 기준 |
| 결함 분류 기준표 | file | 품질특성별 결함 유형 및 등급 분류 기준 |
| 결함 리포트(빈 템플릿) | file | 결함 기록용 빈 양식 샘플 |

## TIP
> RawData는 패치 후 비교의 근거가 되므로 기능별 폴더로 정리하여 저장하세요.
> 메뉴 구조를 따라가며 테스트하면 기능 누락을 줄일 수 있습니다.
