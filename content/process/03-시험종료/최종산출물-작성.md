## 설명
패치 후 RawData 캡처, 기능 리스트 완성, 테스트 케이스 작성 등 최종 산출물을 준비하고 점검한다.

## 체크포인트
- [ ] 패치 후 RawData를 기능별로 캡처했는가? [ref: 패치 후 Rawdata]
- [ ] 기능 리스트를 완성했는가? (인증위 전까지)
- [ ] 기능 리스트 기반 테스트 케이스를 작성했는가? [ref: 테스트케이스]
- [ ] 최종 산출물 점검을 완료했는가? [ref: 패치 전 Rawdata, 패치 후 Rawdata, 패치 전 성능시험 보고서, 패치 후 성능시험 보고서, 패치 전 보안 테스트 보고서, 패치 후 보안 테스트 보고서, 테스트케이스]

## 증빙 예시
- 패치 전/후 RawData (기능별 캡처 이미지)
- 패치 전/후 보안성 테스트 보고서 (Invicti 스캔 결과)
- 패치 전/후 성능 테스트 보고서
- 테스트 케이스

## 합격 기준
모든 최종 산출물이 작성되고 점검이 완료되었다.

## 참조 문서
| 문서명 | 종류 | 설명 |
|--------|------|------|
| 패치 전 Rawdata | file | 패치 전 기능별 캡처 이미지 |
| 패치 후 Rawdata | file | 패치 후 기능별 캡처 이미지 |
| 패치 전 성능시험 보고서 | file | 패치 전 성능 테스트 결과 |
| 패치 후 성능시험 보고서 | file | 패치 후 성능 테스트 결과 |
| 패치 전 보안 테스트 보고서 | file | 패치 전 Invicti 스캔 결과 |
| 패치 후 보안 테스트 보고서 | file | 패치 후 Invicti 스캔 결과 |
| 테스트케이스 | file | 기능 리스트 기반 테스트 케이스 |

## TIP
> 테스트 케이스는 실무상 시험 종료 후 약 1주일 이내에 작성합니다.
> 결함 리포트/회귀 리포트는 별도 관리되며 최종 산출물에는 포함되지 않습니다.
