## 설명
자리·장비 준비 후, 시험 합의서에 명시된 OS 및 필수 SW를 설치하여 제품 설치가 가능한 상태로 구성하는 단계. 합의서의 OS 종류·버전과 필수 SW 목록을 정확히 반영해야 시험 환경의 신뢰성이 확보된다.

## 체크포인트
- [ ] 시험 합의서에 명시된 OS 종류·버전(예: Windows 10 Pro 22H2, Windows Server 2019)을 확인했는가? [ref: 시험 합의서]
- [ ] 기존 장비의 OS가 합의서 요구사항과 일치하는지 확인하고, 새로 설치가 필요한지 판단했는가?
- [ ] IODD 담당자에게 해당 OS 버전의 설치 자료(IODD)를 대여받고, OS 설치를 완료했는가?
- [ ] 합의서에 명시된 필수 소프트웨어(런타임, 프레임워크, DB 클라이언트 등)를 모두 설치했는가? [ref: 시험 합의서]
- [ ] 설치한 OS·SW 정보를 환경구성 시트에 기입했는가? [ref: 환경구성]

## 증빙 예시
- 환경구성 시트 (구글 스프레드 시트, OS 종류·버전 및 설치 SW 목록 기입)
- OS 설치 완료 화면 캡처 (시스템 정보 → OS 버전 확인)
- IODD 대여/반납 기록

## 합격 기준
1. 합의서 요구사항에 맞는 OS와 필수 SW가 모두 설치되어 제품 설치가 가능한 상태이다.
2. PL이 공유한 '환경구성'파일에 설치 정보가 모두 입력

## TIP
> OS를 새로 설치할 경우, Windows Update를 통해 최신 보안 패치까지 적용하세요.
> 합의서에 명시되지 않은 SW라도, 제품 설치 매뉴얼에서 요구하는 런타임(예: .NET, Java, Node.js)은 함께 설치해야 합니다.
