## 설명
PL로부터 시험 배정 메일 수신 후, 시험 합의서를 기반으로 시험 번호·일정·환경 요구사항 등 핵심 정보를 파악하고 분석하는 단계. 이 단계에서 확인한 정보가 이후 환경 구성의 범위와 일정 관리의 기준이 된다.

## 체크포인트
- [ ] 세부 시험 일정(결함 리포트 작성일, 패치일 등)에 대해 조율되었는가? [ref: 시험 합의서]
- [ ] 사용자 매뉴얼, 제품 설명서를 업체로부터 수령하고, 제품명·버전이 시험 합의서와 일치하는지 확인했는가? [ref: 사용자 매뉴얼, 제품 설명서]
- [ ] 시험환경 요구사항(서버 유무, 필요 장비 수, OS 종류·버전, 하드웨어 사양, 네트워크 환경)을 파악하고, 이후 환경 구성 범위를 산정했는가? [ref: 시험 합의서]

## 증빙 예시
- 시험 합의서 (PDF/출력물)
- 시험 배정 메일 (PL 발송)
- 사용자 매뉴얼 (업체 제공)
- 제품 설명서 (업체 제공)

## 합격 기준
1. 시험 정보(시험 번호, 시험 일정(시작/종료일, 1/2차 결함, 패치일 등))가 모두 입력
2. 제품 정보(시험 합의서, 사용자 매뉴얼, 제품 설명서)가 모두 입력
3. 시험 합의서 내용(시험대상 제품, 시험환경(서버 유무, 필요 장비 수, 운영체제, 하드웨어 사양, 네트워크 환경, 기타 환경)을 모두 입력

## 참조 문서
| 문서명 | 종류 | 설명 |
|--------|------|------|
| 시험 합의서 | file | 시험 범위와 조건이 명시된 합의서 |
| 사용자 매뉴얼 | file | 제품 사용법이 기술된 매뉴얼 |
| 제품 설명서 | file | 제품 기능과 사양이 기술된 설명서 |

## TIP
> 시험 합의서 PDF를 업로드하면 자동으로 핵심 정보를 추출합니다. 자동 추출이 부정확한 경우 직접 수정할 수 있습니다.
> 합의서의 시험환경 항목을 꼼꼼히 확인해야 SETUP-02~05 환경 구성 시 누락이 없습니다.
> 제품 유형(웹/CS)에 따라 보안 테스트(Invicti) 수행 여부가 달라지므로 이 단계에서 미리 파악하세요.
